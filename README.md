# Локализация AntiRaidBot

Открытый репозиторий переводов для **AntiRaidBot**. Здесь хранятся тексты интерфейса бота: сообщения команд, кнопки, меню, ошибки, описания бейджей и другие строки.

Бот загружает переводы из этого репозитория автоматически. После коммита в `main` новые тексты применяются на запущенном боте без перезапуска, обычно в течение минуты.

## Структура

```text
localization/
├─ lang.json
├─ locale/
│  ├─ RU-ru.json
│  └─ EN-us.json
└─ README.md
```

`lang.json` описывает языки, которые бот показывает в меню `/language`:

```json
{
    "languages": [
        {
            "code": "RU-ru",
            "aliases": ["ru", "ru-RU", "RU-ru"],
            "label": "Русский",
            "flag": "🇷🇺",
            "description": "Установить русский язык для бота"
        }
    ]
}
```

Поля:

- `code` — имя JSON-файла без `.json`, например `RU-ru` для `locale/RU-ru.json`.
- `aliases` — дополнительные варианты кода языка, которые бот тоже будет понимать.
- `label` — название языка в меню.
- `flag` — флаг или emoji для меню.
- `description` — описание пункта в меню выбора языка.

Каждый файл в `locale/` — это JSON-объект:

```json
{
    "language.title": "Язык сервера",
    "general.buttons.support": "Поддержка"
}
```

Ключи должны совпадать между языками. Если в переводе нет нужного ключа, бот попробует взять английский текст, затем русский, а если ключ не найден нигде — покажет сам ключ.

## Как исправить перевод

1. Откройте нужный файл в `locale/`, например `RU-ru.json`.
2. Найдите строку по ключу или по тексту.
3. Исправьте только значение справа от ключа.
4. Проверьте, что JSON остался валидным: запятые, кавычки и переносы строк важны.
5. Отправьте Pull Request с кратким описанием изменения.

Пример:

```json
"general.buttons.support": "Сервер поддержки"
```

## Как добавить новый язык

1. Скопируйте `EN-us.json`.
2. Переименуйте копию в код языка, например `DE-de.json`, `FR-fr.json`, `UK-ua.json`.
3. Переведите все значения, не меняя ключи.
4. Добавьте язык в `lang.json`.
5. Не удаляйте плейсхолдеры в фигурных скобках: `{guild_name}`, `{user_name}`, `{count}` и похожие.
6. Создайте Pull Request.

После обновления `lang.json` язык появится в меню `/language` автоматически, без перезапуска бота. Если языков станет больше 25, Discord не позволит показать их все в одном select-меню; в таком случае напишите нам на сервер поддержки, и мы расширим интерфейс выбора.

## Правила перевода

- Не меняйте имена ключей.
- Не удаляйте Markdown-разметку, если она нужна в строке: `**жирный текст**`, `` `код` ``, ссылки.
- Сохраняйте Discord-эмодзи и кастомные эмодзи в формате `<:name:id>`.
- Сохраняйте плейсхолдеры без перевода: `{permissions}`, `{timestamp}`, `{role_name}`.
- Если строка содержит `\n`, это перенос строки. Его можно оставлять как есть.
- Пишите естественно и понятно, без машинного дословного перевода.

## Проверка JSON

Перед Pull Request можно проверить файлы командой:

```bash
python -m json.tool locale/RU-ru.json
```

Для проверки всех локалей:

```bash
python - <<'PY'
import json
from pathlib import Path

json.loads(Path("lang.json").read_text(encoding="utf-8"))
print("lang.json: ok")

for path in Path("locale").glob("*.json"):
    json.loads(path.read_text(encoding="utf-8"))
    print(f"{path}: ok")
PY
```

## Когда изменения попадут в бота

После merge или прямого коммита в ветку `main` бот сам проверит репозиторий и обновит тексты без перезапуска. Интервал проверки задаётся в боте переменной `LOCALIZATION_REFRESH_SECONDS`, по умолчанию — 60 секунд.

Если изменение не появилось сразу, подождите немного. Если текст всё равно не обновился, напишите на сервер поддержки.
